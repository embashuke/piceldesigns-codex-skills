# Visual Reference Translator

## Purpose

Use this skill before building or reviewing Piceldesigns frontend components and pages.

The skill translates approved visual references into practical implementation guidance for:
- typography hierarchy
- spacing rhythm
- header restraint
- selected-work layout rhythm
- project card treatment
- service card restraint
- page visual hierarchy
- negative space
- image-led credibility
- component polish
- visual QA

This skill does not create page structure.

This skill does not override approved architecture.

This skill does not invent new sections, pages, components, CTAs, or proof claims.

## Source hierarchy

Before visual interpretation, check the approved project documents in this order:

1. Frozen Website Architecture v2.2
2. CMS Schema / Content Model Map Final Locked
3. Developer Execution Brief v3
4. Website Copy Draft v2.2
5. UI Direction Pack v5

If a conflict appears, higher authority wins.

## Approved visual references

Use only the approved visual references from the Developer Execution Brief.

Approved references:
- Alphamark — alphamark.design
- Symbol Studio
- The First The Last — Works page only
- Saffron / Markel — later-phase case study thinking only

Do not broaden the reference stack without approval.

## Borrow rules

From Alphamark, borrow:
- minimal header restraint
- strong typographic hierarchy
- tracked uppercase project/category labels
- restrained selected-work grid structure

Reject from Alphamark:
- award badge clutter
- rating widgets
- decorative proof clutter

From Symbol Studio, borrow:
- quiet confidence
- understated studio presence
- work-led credibility
- limited explanatory noise

Reject from Symbol Studio:
- Behance/platform UI elements
- follower metrics
- social-platform clutter

From The First The Last Works page, borrow:
- work grid rhythm
- image-led project presentation
- visual editing quality
- confident use of project imagery

Reject:
- anything outside the Works page
- layout expansion not approved by Piceldesigns architecture

From Saffron / Markel, borrow only:
- future case study thinking
- client/context/services/outcome editorial structure for later phases

Reject:
- Phase 1 case study detail pages
- case-study page builds
- new project detail routes

## Piceldesigns visual direction

The website should feel:
- serious
- restrained
- clear
- commercially credible
- design-studio appropriate
- image-led
- typographically strong
- calm
- selective
- not noisy

Avoid:
- generic agency templates
- SaaS landing page energy
- startup dashboard aesthetics
- cluttered testimonial/proof sections
- excessive gradients
- decorative animations
- centered paragraph blocks
- over-designed cards
- fake luxury treatment
- social media platform clutter

## Implementation translation

When invoked before component work, produce:

1. Visual Reference Translation Table
2. Component-level guidance
3. Page-level guidance
4. Borrow / reject reminders
5. Visual risks
6. Acceptance criteria

## Component guidance must cover

- Header/Nav
- Footer
- Page Hero
- Service Hero
- CTA Block
- Service Card
- Project Card
- Proof/Trust Strip
- Contact Form
- Work Grid
- 404 Page

## Page guidance must cover

- Home
- Services
- Service detail pages
- Work
- About
- Contact

## Visual QA checklist

Check:
- Does the page feel restrained?
- Is typography leading the layout?
- Is there enough negative space?
- Are cards too busy?
- Are CTAs clear but not excessive?
- Is WhatsApp still dominant where required?
- Are project visuals doing enough credibility work?
- Is there any generic agency fluff?
- Is there any SaaS/product-template energy?
- Are visual references being copied too literally?
- Are rejected reference elements creeping in?

## Hard stops

Stop and ask for approval if:
- a new visual reference is introduced
- a reference suggests an unapproved layout
- visual direction conflicts with the approved architecture
- Codex tries to create new components to match inspiration
- Codex tries to copy another site too closely
- Codex tries to add awards, badges, ratings, testimonials, stats, or social proof not approved
- Codex tries to build case study detail pages in Phase 1
